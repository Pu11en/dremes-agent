# Client Site — Populate Convention

When the user says "populate the client dashboard" or "update the brand site" or
"sync ads to the website", follow this exact workflow. This populates the Drew Pullen
client showcase site at `github.com/Pu11en/drewpullen-site` (deployed on Vercel —
any push to main auto-deploys).

## WHAT THIS SKILL DOES

Copies approved ad images from the Dremes Railway volume into the static client site
repo and rebuilds the ad manifest JSON. The client site then shows those ads on the
`/brands` page under each brand's section.

## PREREQUISITES

- Write access to `github.com/Pu11en/drewpullen-site`
- The dremes-agent Railway deployment must be running (volume at `/data/refs`)
- `git` configured with push access to the client site repo

## POPULATE WORKFLOW

### 1. Clone the client site repo

```
git clone git@github.com:Pu11en/drewpullen-site.git /tmp/drewpullen-site
cd /tmp/drewpullen-site
```

If already cloned, `git pull` to get latest.

### 2. Read approved ads from the Railway volume

For each brand, the approved ads live at:
```
/data/refs/output/ad-approval/{brand}.json
```

The JSON format is:
```json
{
  "pending": ["file1.png", "file2.png"],
  "approved": ["island-splash_20260501_120000_001.png"],
  "bad": []
}
```

Only copy ads from the `"approved"` array.

### 3. Copy PNG images

For each approved ad filename:
```
cp /data/refs/output/{filename}.png  public/ads/{brand}/{filename}.png
```

Create the destination directory if needed:
```
mkdir -p public/ads/{brand}/
```

> Note: All content files in this repo live under `public/` because Vercel
> is configured with `public/` as the output directory.

### 4. Read sidecars for product names

For each approved ad, read its sidecar for product metadata:
```
/data/refs/output/{filename}.instructions.txt
```

Extract the `PRODUCTS:` line to get product names. Map to the correct product
display name from the brand config.

### 5. Rebuild the ad manifest JSON

Write `public/data/ads/{brand}.json` with this format:
```json
[
  {
    "filename": "island-splash_20260501_120000_001.png",
    "product_name": "Mango Passion"
  }
]
```

Only include ads whose image files exist in `public/ads/{brand}/`. Never include
entries with missing images.

### 6. Commit and push

```
git add data/ads/ public/ads/
git commit -m "populate: {brand} — {N} approved ads"
git push origin main
```

Vercel auto-deploys on push. The site updates within ~30 seconds.

```

├── vercel.json              ← Routes /brands → brands.html

    ├── index.html           ← Landing page (Drew Pullen, book a call)

    ├── drew-pullen-resume.pdf
    ├── data/ads/
    │   ├── island-splash.json  ← Ad manifest (generated by this skill)

        ├── island-splash/      ← PNG images (copied from Railway volume)
        └── cinco-h-ranch/      ← PNG images (copied from Railway volume)
```

## ANTI-PATTERNS

- Do NOT copy non-approved ads — only read from the `"approved"` array in approval JSON
- Do NOT modify `index.html` or `brands.html` unless asked — this skill only touches data and images
- Do NOT remove ad entries from the manifest unless the corresponding image file is also removed
- Do NOT leave the git repo in a dirty state — commit everything or nothing
- Do NOT push to the dremes-agent repo for this — always push to drewpullen-site
